# 5分デモ台本

## デモの目的

Supply Sentinel が、外部リスクを検知し、自社データと照合し、初動対応を準備する流れを5分で見せる。

## 0:00 - 0:30 課題提示

話すこと:

> 原材料不足は、ニュースが出た瞬間に問題になるのではなく、自社のどの製品・顧客・工場に影響するか分からない時間が問題になります。

見せるもの:

- ナフサ供給不安ニュース
- サプライヤ通知
- 在庫/BOM/受注データ

## 0:30 - 1:00 エージェント起動

話すこと:

> Supply Sentinel は Azure Functions の Timer Trigger で定期巡回します。常時監視しているように見えますが、実装はシンプルな定期実行です。

見せるもの:

- 実行ログ
- 定期実行フロー

## 1:00 - 1:45 リスク抽出

話すこと:

> Azure OpenAI がニュースとサプライヤ通知から、対象原材料、遅延、供給制限、期間、根拠を抽出します。

見せるもの:

```json
{
  "material": "naphtha",
  "risk_type": "supply_delay",
  "delay_days_min": 5,
  "delay_days_max": 7,
  "allocation_rate_percent": 70,
  "severity": "high"
}
```

## 1:45 - 2:45 自社影響評価

話すこと:

> ここがSupply Sentinelの核です。外部リスクを、自社のBOM、受注、在庫に照らして、業務影響に変換します。

見せるもの:

```text
影響製品: Resin A, Solvent B
影響顧客: Customer Alpha, Customer Beta
影響工場: Chiba Plant
最短在庫残日数: 5日
```

## 2:45 - 3:30 リスクスコアと初動案

話すこと:

> AIだけで危険判定するのではなく、在庫残日数、顧客優先度、代替材の有無を計算してスコア化します。

見せるもの:

```text
リスクスコア: 82/100
危険度: High

推奨初動:
- サプライヤへ割当数量を確認
- 優先顧客向け在庫を確保
- 代替材 NAP-ALT-01 の適用可否を確認
- 顧客説明用の一次案を準備
```

## 3:30 - 4:20 Teams通知とレポート

話すこと:

> 現場にはTeamsで短く通知し、管理職には判断用レポートを生成します。

見せるもの:

- Teams風アラート
- 管理職向けレポート

## 4:20 - 5:00 Human-in-the-loop

話すこと:

> Supply Sentinel は発注変更やサプライヤ切替を勝手に実行しません。AIは判断材料を揃え、重要判断は人が承認します。

締め:

> 従来は自社内で起きた問題を最適化していました。Supply Sentinel は、自社外で起き始めた危機を自社影響に翻訳し、現場の初動を早めます。

## デモ時に避ける表現

- 完全予測
- 自動発注
- 株価予測
- サプライヤ自動切替
- 何でも監視できる

## デモ時に使う表現

- 予兆検知
- 早期警戒
- 自社影響への翻訳
- 初動対応の高速化
- 人が承認するAI
